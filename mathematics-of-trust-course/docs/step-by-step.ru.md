> **Для кого:** разработчики, строящие доверие агентов, репутацию и аудит системного риска на оракулах AIMarket.  
> **Язык лаб:** `COURSE_LANG=en|ru|es` · строки UI в `i18n/` · этот гайд на русском.  
> **English:** [step-by-step.md](./step-by-step.md) · **Español:** [step-by-step.es.md](./step-by-step.es.md)

## Зачем этот курс

Мультиагентные системы постоянно отвечают на один вопрос: *какому агенту (или агрегату, или маршруту) я доверяю — и могу ли я это доказать?* Курс превращает этот вопрос в математику, которую можно запустить. Пять живых оракулов AIMarket считают сигналы — **LUMEN** (PageRank/EigenTrust), **PERCOLA** (пороги перколяции), **FOURIER** (спектральная связность), **MURMURATION** (устойчивый консенсус) и **ABLATION** (каскадный риск кучи песка) — и каждый сигнал приходит со своим **сертификатом проверки**, так что верить оракулу на слово не нужно. Лабы детерминированы и работают на NumPy: для базового трека не нужны ни API-ключи, ни сеть.

К концу вы умеете оценивать граф «кто кому доверяет», находить ключевой узел, удаление которого раскалывает граф, измерять, насколько граф близок к расколу, агрегировать шумные прогнозы против злоумышленника и оценивать риск хвоста системного каскада — а затем складывать всё это в один проверяемый аудит доверия.

## Выбор трека

| Трек | Модули | Время | Оракулы |
|------|--------|-------|---------|
| **Базовый** | M1 → M2 → M3 → M4 | ~2 ч | LUMEN, PERCOLA, FOURIER, MURMURATION |
| **Продвинутый** | Базовый + M5 каскад | +30 мин | + ABLATION (куча песка) |
| **Итоговый** | M6 сводный аудит | +20 мин | LUMEN + FOURIER + PERCOLA вместе |

Лаб **шесть** (`labs/lab01`–`lab06`), по одной на модуль. Каждая печатает блок `Trace` (трейс) — это её журнал аудита. Лабы с проверяемым оракулом печатают ещё и `verify: True` — результат независимо перепроверен, а не просто заявлен.

## M1 — EigenTrust на LUMEN

**Концепция:** стационарное распределение PageRank на графе «кто кому доверяет».  
**Лаба:** `labs/lab02_trust_pagerank.py` · **~20 мин** · нужна установка

### Шаги

1. `python labs/lab02_trust_pagerank.py`
2. Посмотрите учебный граф в `courselib/trust.py` — `demo_trust_graph()`.
3. Ранжированные оценки: `hub` и хорошо связанные агенты должны быть выше периферии.
4. Загляните в `oracles/oracles/lumen/lumen/pagerank.py` — ядро EigenTrust (5 мин).
5. **Упражнение:** `python labs/run_exercises.py --module m1`.

### Самопроверка

- [ ] Оценки в сумме дают 1 (распределение вероятностей)
- [ ] Транзитивное доверие поднимает узлы, которым доверяют доверенные узлы

## M2 — Перколяция и ключевые узлы

**Концепция:** целенаправленное удаление узлов и критические пороги на графах доверия.  
**Лаба:** `labs/lab01_keystone_nodes.py` · **~20 мин** · нужна установка

### Шаги

1. `python labs/lab01_keystone_nodes.py`
2. Следите за `f_c` — критической долей узлов, удаление которых обрушивает связность.
3. Изучите `keystones` (ключевые узлы) — мосты между двумя кликами (`3`, `4`, `8` в учебном графе).
4. Подтвердите `verify: True` — PERCOLA пересчитывает порог, не доверяя оракулу на слово.
5. **Упражнение:** `python labs/run_exercises.py --module m2`.

### Самопроверка

- [ ] Понимаете, почему мостовой узел — ключевой
- [ ] Проверка проходит без повторного полного перебора Монте-Карло

**Лаба:** `labs/lab03_spectral_cut.py` · **~20 мин** · нужен M1 или M2

5. **Упражнение:** `python labs/run_exercises.py --module m3`.

- [ ] Сертификат собственной пары Фидлера проверяется за O(E)

## M5 — Каскадный риск

**Концепция:** лавины абелевой кучи песка и степенной риск хвоста.  
**Лаба:** `labs/lab05_sandpile_cascade.py` · **~30 мин** · нужны M1–M4

- [ ] Размеры лавин не зависят от порядка (абелево свойство)

## M6 — Итоговый проект: математика доверия

**Концепция:** объединить сигналы PageRank, спектра и перколяции в один проверяемый аудит доверия.  
**Лаба:** `labs/lab06_trust_audit.py` · **~20 мин** · нужны M1–M3

### Шаги

1. `python labs/lab06_trust_audit.py`
2. Прочитайте однострочный аудит по `demo_trust_graph()`:
   - **Самый доверенный** узел и его оценка PageRank (LUMEN) — ожидается `hub`.
   - **Алгебраическая связность** λ₂ и её флаг `verify` от FOURIER.
   - **Порог перколяции** f_c и его флаг `verify` от PERCOLA.
   - **Ключевые узлы**, связывающие граф мостами.
3. Подтвердите итоговый `verify: True` — это **И** (AND) сертификатов спектра и перколяции. Одно проваленное доказательство проваливает весь аудит.
4. Откройте `courselib/trust.py` — `trust_audit_summary()`: вызывает `score_pagerank`, `analyze_spectral` и `analyze_percolation`, затем возвращает `top_trusted`, `fiedler_value`, `percolation_f_c`, `keystones` и оба флага проверки.
5. Набросайте, как бы вы пропускали действие агента сразу по трём сигналам (доверенный источник **и** связный граф **и** отсутствие хрупких ключевых узлов).
6. **Упражнение:** `python labs/run_exercises.py --module m6`.

### Ожидаемый вывод

- Строка `== Trust math capstone ==`
- `Trust audit (7 nodes)` с самым доверенным = `hub`, λ₂, f_c и ключевыми узлами
- Итоговый `verify: True`

### Самопроверка

- [ ] Называете три оракула, из которых собран аудит (LUMEN, FOURIER, PERCOLA)
- [ ] Понимаете, почему аудит проходит только когда проходит *каждый* сигнал

## Упражнения и сертификат

После лаб запустите проверки:

```bash
python labs/run_exercises.py
python labs/run_exercises.py --certificate "Иван Петров" --lang ru
```

| Модуль | Что проверяет упражнение |
|--------|--------------------------|
| M1 | Оценки PageRank в сумме дают 1 |
| M2 | Ключевые узлы найдены; проверка PERCOLA проходит |
| M3 | Сертификат Фидлера проверяется |
| M4 | Консенсус устойчив к выбросу |
| M5 | τ каскада и суммарное число обрушений проверяются |
| M6 | Сводный аудит: `hub` самый доверенный, доказательства спектра и перколяции проходят |

Сертификат покрывает все шесть модулей. HTML открывается в браузере; **Print → Save as PDF** для версии, которой можно поделиться. ID сертификата — `sha256(имя|дата)[:12]`, поэтому одно и то же имя в один и тот же день всегда даёт один и тот же ID. Чтобы перегенерировать без перезапуска проверок, добавьте `--skip-check`; путь задаётся через `-o path/to/certificate.html`.

## Наблюдаемость в каждой лабе

**Концепция:** каждая лаба пишет решения в `Trace` (трейс) — это ваш журнал аудита событий доверия.

### Шаги (после каждой лабы)

1. Прокрутите к секции **trace** в конце вывода.
2. Сопоставьте события со stdout: оценки доверия, пороги перколяции, спектральные значения, статистики консенсуса, метрики каскада.
3. В M5 свяжите события `cascade` с напечатанными τ и CVaR.

- [ ] Трейс — это «превью SIEM» для математики доверия

## Типичные проблемы

| Проблема | Решение |
|----------|---------|
| `ModuleNotFoundError: courselib` | Запуск из корня репозитория; лабы добавляют родителя в `sys.path` |
| `ImportError: Oracle package 'lumen'` | Установите срез оракулов или запускайте из монорепо aicom |
| Нет `numpy` | `pip install -e ".[oracles,dev]"` |
| Пустые строки RU/ES | `export COURSE_LANG=ru`; ключи `i18n/ru.json` должны совпадать с `en.json` |
| Падает i18n в pytest | Добавьте недостающие ключи во все три JSON |
| Проверка PERCOLA не проходит | `samples` должен совпадать в analyze и verify |
| Старый код в Colab | Перезапустите setup-ячейку; закрепите ветку `main` |
